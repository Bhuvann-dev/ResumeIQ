# ResumeIQ — API Contract

**Status:** v0.1 (MVP slice) · **Base URL (local):** `http://localhost:8000`

This document reflects the endpoints that actually exist in [`api/`](../api). It
grows as endpoints are built — the async/job-status and export endpoints in
[architecture.md](architecture.md) are not implemented yet and are marked
_planned_.

Interactive docs (auto-generated by FastAPI) are served at `/docs` when the API
is running.

---

## `GET /health`

Liveness + whether the AI key is configured.

**200 OK**
```json
{ "status": "ok", "ai_configured": true }
```

---

## `POST /analyze`

Upload a resume (and optionally a job description) and get a synchronous ATS
analysis.

**Request** — `multipart/form-data`

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `file` | file | yes | PDF or DOCX, max 5 MB |
| `job_description` | text | no | Paste a JD to get `jd_match_percent` + `missing_keywords` |

**200 OK** — `AnalysisResult`
```json
{
  "ats_score": 82,
  "summary": "Solid foundation, but missing key keywords and quantified impact.",
  "dimension_scores": { "format": 88, "keywords": 55, "content": 74, "structure": 92 },
  "detected_role": "SDE Fresher",
  "jd_match_percent": 71,
  "missing_keywords": ["REST API", "Git"],
  "issues": [
    {
      "category": "critical",
      "dimension": "keyword",
      "title": "Missing core keywords from the job description",
      "message": "The JD emphasizes 'REST API' and 'Git', which don't appear in your resume. ATS keyword filters may screen you out.",
      "suggestion": "Add these to your Skills section and weave them into a project bullet where you genuinely used them."
    }
  ]
}
```

**Field reference**

| Field | Type | Description |
|-------|------|-------------|
| `ats_score` | int (0–100) | Overall ATS readiness |
| `summary` | string | One–two sentence verdict |
| `dimension_scores` | object | `format`, `keywords`, `content`, `structure` — each 0–100 |
| `detected_role` | string | Inferred target role |
| `jd_match_percent` | int \| null | Present only when a JD was supplied |
| `missing_keywords` | string[] | JD keywords absent from the resume |
| `issues[]` | array | Categorized findings |
| `issues[].category` | enum | `critical` \| `warning` \| `suggestion` |
| `issues[].dimension` | enum | `format` \| `keyword` \| `content` \| `structure` \| `contact` |
| `issues[].title` | string | Short label |
| `issues[].message` | string | What's wrong and why it matters |
| `issues[].suggestion` | string | How to fix it |

**Errors**

| Status | Meaning |
|--------|---------|
| 400 | Empty file |
| 413 | File exceeds 5 MB |
| 422 | Unreadable file (e.g. scanned image, corrupt) |
| 502 | AI/provider failure |
| 503 | Server missing `ANTHROPIC_API_KEY` |

Error bodies follow FastAPI's shape: `{ "detail": "<message>" }`.

## `POST /improve`

Upload a resume (and optional JD) and get an AI-rewritten, ATS-friendly version.

**Request** — `multipart/form-data` (same fields as `/analyze`).

**200 OK** — `ImproveResult`
```json
{
  "improved_markdown": "# Jane Doe\njane@example.com · github.com/jane\n\n## Summary\n...\n\n## Skills\n- Python\n- React\n",
  "key_changes": [
    "Rewrote experience bullets with strong action verbs",
    "Added a Skills section with JD-relevant keywords",
    "Removed the two-column layout that broke ATS parsing"
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `improved_markdown` | string | Full rewritten resume in simple markdown (`#` name, `##` sections, `-` bullets) |
| `key_changes` | string[] | Summary of the main improvements |

Same error codes as `/analyze` (400 / 413 / 422 / 502 / 503).

---

## `POST /export`

Render a resume (markdown, optionally edited by the user) to a downloadable,
ATS-friendly **DOCX or PDF**. The PDF keeps URLs/emails as clickable links.

**Request** — `application/json`
```json
{ "markdown": "# Jane Doe\n...", "filename": "resume_improved", "format": "pdf" }
```

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `markdown` | string | — | Resume in markdown (the edited text from the UI) |
| `filename` | string | `resume_improved` | Sanitized to a safe basename server-side |
| `format` | `"docx"` \| `"pdf"` | `docx` | Output format |

**200 OK** — binary file with a `Content-Disposition: attachment` header. Content-Type is `application/pdf` or the DOCX media type accordingly.
